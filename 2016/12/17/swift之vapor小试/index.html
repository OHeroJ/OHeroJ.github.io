swift之vapor小试

Veröffentlicht am 2016-12-17   |  

零零碎碎的学了下 swift 后台方面的东西，16年年末，反观前些年，自己在编程语言的建树，可谓杂多广亦零零碎碎，未深入，有些也就是庭前看花，浅尝辄止。17年即将到来，年前新也给自己博客定个风格套路，避免写出的东西也零碎不堪。

此博文来谈 Vapor,纵观 github ,swift web 框架 Prefect 居首位。然每个人挑选框架亦与其个人喜好关联。随性的人喜欢按照自己的兴趣去选择。我并没有选择 Prefect：

  1. 不优雅
  2. 传统
  3. 企业性质
  4. 不是我关注的第一个 swift 的后台框架

如果你想去了解现有框架的对比，可移步到 这里

多说无益，毕竟说得越多（文笔不好，现在还是多写点，多暴露一点自己的问题，勿怪），就会暴露更多的问题，就如我们写代码，写的越多，人家 review 的越多，然后打回的几率也大，所以代码的简洁和优化是多么的重要

牛逼的程序员是思考如何将代码越写越少而不是越写越多！

知识点之前，说说最近的一些感触

最近感觉自己在写代码的时候，自由度被限制了。公司现在在不断的要求 代码规范和代码提交规范，git 管理规范，文档规范…, 代码日产果断是降低不少。但过了这么段苦逼的日子，看看现在自己的代码确实欣慰少许。细节方面略过，如果你以后是做规范的，我觉得可以往这几个方面：

  1. 代码方面：

    注释风格 | 空行 | 空格 | 命名 | 社区比较认定的规范(从 github 上找)

  2. 代码提交方面(git):

    提交有意义的代码 | 提交规范 | 分支命名规范 | 分支管理规范 | 审核规范

  3. 文档方面(markdown):

    这块我局的大家把文档质量保证，其实排版问题不到，前提说的你用的是 Markdown

    以上几点非本文重点，不在过多阐述，等我在多点经验整理获取更具合理性
