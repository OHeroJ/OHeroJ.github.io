《在修改代码的艺术》中有这么一段话

最为强大的特性添加技术便是测试驱动开发(TDD)

驱动测试开发的过程：

设想有这么一个方法，能够帮我们解决问题的某个部分; 接下来我们为这个设想中的方法编写一个失败的测试用例。此时方法尚未存在，
但既然我们能够为它编写测试，我们就对接下来要编写的代码做什么事情有一个确定的认识。

TDD 算法

  1. 编写一个失败测试用例
  2. 让它通过编译
  3. 让测试通过
  4. 消除重复
  5. 重复上述步骤

提醒

测试驱动开发的它使得我们可以在同一个时间之关注一件事情，要么在编码，那么在重构
在编写一些新代码之后，我们便可以通过重构来消除新旧代码之间的任何重复。

在遗留代码的工作场景下，步骤可扩展

  1. 将想要的修改的类置于测试之下
  2. 编写一个失败的测试用例
  3. 让它通过编译
  4. 让测试通过（尽量不要改动既有代码）
  5. 消除重复
  6. 重复上述步骤
